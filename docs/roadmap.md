# Roadmap

## 总体目标
- 形成可回放、可观测、可测试的会议引擎。
- 让会议房间具备稳定的状态机与权限策略。
- 逐步补齐工程化能力：测试、文档、规范。

## 里程碑

### Phase 1（当前）
- 会议引擎模块拆分（协议/格式化/统计）。
- 事件溯源与回放能力可演示。
- 基础单测覆盖会议引擎核心逻辑。

### Phase 2
- 扩展事件维度统计（按角色/操作类型/时间窗）。
- 会议回放面板增强：摘要、筛选、导出。
- 关键流程异常处理与恢复策略。

### Phase 3
- 会议房间稳定性提升（断线恢复/设备切换策略）。
- 业务协作产出（纪要/待办）能力落地。
- E2E 流程级用例覆盖核心用户路径。

## 风险与依赖
- 事件量上升导致存储压力，需要分页/归档策略。
- 若未来接入后端，需统一事件协议与鉴权校验。
- 多人实时通话需要引入信令与媒体服务。
